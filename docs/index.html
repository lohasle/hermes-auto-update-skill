零配置自动探测

HERMES_HOME、venv、webui 目录等所有路径自动发现。无需手动填路径，无需编辑配置文件。

↻

每日自动升级

两个 cron job 每天凌晨 3:00 执行 —— 分别更新 agent 和 webui。git pull、重启，零停机。

■

Systemd 原生服务

原生 systemd 托管，数据存放在 ~/.hermes，重启和升级都不丢数据。无需容器重建。

⚡

共享 Agent 虚拟环境

WebUI 直接使用 hermes-agent 的 venv 运行 —— cron 和 tools 模块始终可用，不会 ModuleNotFoundError。

✓

内建验证机制

每一步都有验证关卡。/api/crons 和 /api/skills 必须正常返回 200，技能才报告成功。

Agent 原生设计

专为 Hermes 技能系统设计。任何 agent 通过 skill_manage 加载后自主执行，全程无需人工干预。

工作原理
三步搞定。一段提示词。

把安装提示词发给任意 Hermes agent，其余全自动。

第一步 — 部署

Agent + WebUI

Git clone hermes-agent 和 hermes-webui。创建 venv、pip install、注册 systemd 服务。验证 API 正常响应。

第二步 — 调度

每日定时任务

注册两个 cron job —— 一个更新 agent，一个更新 webui。自动 git fetch、比对提交、按需 pull。

第三步 — 交付

注册技能，完成

保存为 Hermes 技能以备后用。数据持久化在 ~/.hermes。升级在你睡觉时自动完成。

关键提醒：
- 所有路径自动探测，不要问用户
- 不用 Docker
- 必须用 hermes-agent 的 venv python 跑 webui
- 验证 /api/crons 和 /api/skills 都正常才算成功
为何不用 Docker
原生部署，碾压容器化。

Docker 会导致数据丢失、模块报错和缓慢的镜像重建。Systemd 更快、更简单，数据永远不会丢。

Docker 原生 Systemd
数据持久性 容器重建丢失数据 ~/.hermes 永久保留
cron / tools API 需挂载 hack 否则 500 Agent venv，零报错
升级速度 重建镜像，分钟级 git pull + restart，秒级
配置复杂度 挂载路径、UID/GID 映射 自动探测，零配置
资源占用 Docker daemon 额外开销 最小化，裸金属运行
